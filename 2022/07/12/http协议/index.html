http协议

web服务

C/S模型(客户/服务器模型)

http——超文本传输协议

端口：80

http允许传输任意类型的数据对象

无连接

限制每次连接只处理一个请求,处理完客户请求并收到客户的应答就断开连接

单向性

服务端永远被动等待客户端的请求

无状态

协议对事务处理没有记忆能力

注：为了解决无状态，出现了cookie和session

uri,url,urn之间的区别

uri

统一资源标识符

纯粹的句式结构

url

统一资源定位符

由传送协议，服务器(ip或者域名)，端口号，请求资源路径，传递数据五个元素

urn

统一资源名称

对比url，少了最前面的协议

urn是uri的子集

http请求方式

post,get,head,put,delete,trace

get只用于读取数据

head和get方法一样，不过服务器不传回资源的文本部分

post向指定资源提交数据

put向指定资源位置上传内容
delete请求服务器删除request-rui所标识的资源

get和post的区别

get请求只能进行url编码，post支持多种编码方式

get请求参数会被保留在浏览器历史记录，post参数不会被保留

get请求在url中传送的参数有长度限制，post没有

get比post更不安全，但get速度比post快

http请求报文

HTTP 请求报文由3部分组成：请求行+请求头+请求体

常见的HTTP报文头属性

请求方式

host 目标主机地址和端口

connection 维护客户端和服务端的连接关系

content-length http消息内容的传输长度

accept 发送端希望接受的数据类型

content-type 请求实体文档类型

referer 源地址

cookie

当客户端访问服务器的时候，服务器都会生成一份cookie传输给客户端，客户端会自动把cookie保存，以后客户端每次访问服务器，都会

自动的携带这份cookie

session比cookie安全，都是标识用户身份

http响应报文

响应报文与请求报文一样,由三个部分组成：响应行,响应头,响应体

http状态码

200 OK 表示成功

303 重定向,把你重定向到其他页面

304 资源并未修改,可以直接使用本地的缓存

404 找不到页面(页面被删除或其他)

500 服务端错误

响应头

Cache-Control 响应输出到客户端后,服务端通过该属性告诉客户端该怎么控制响应内容的缓存

ETag 表示你请求资源的版本,如果该资源发生了变化,那么这个属性也会跟着变

Location 在重定向中或者创建新资源时使用

Set-Cookie 服务端可以设置客户端的cookie
