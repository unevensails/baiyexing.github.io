老规矩，先arp扫存活主机


发现80端口开放，进行访问



得知这次只有一个flag，我们只需要拿到root权限就能拿到flag
由于没有任何提示，我们先尝试用dirsearch扫描目录，发现readme.txt

访问readme.txt，可以知道该网站cms为joomla

尝试用msf攻击


试了几个漏洞都没啥用，这时候看wp才知道要用searchspliot

“searchsploit”是一个用于Exploit-DB的命令行搜索工具，可以帮助我们查找渗透模块。
Exploit-DB是一个漏洞库，Kali Linux中保存了一个该漏洞库的拷贝，利用上面提到的命令就可以查找需要的渗透模块，它将搜索所有的漏洞和shellcode而且该漏洞库是保存在本地的，在没有网络的情况下也可以使用。
基本搜索方法就是searchsploit+可能包含漏洞的软件/系统等等，对应回显存在的漏洞和用于渗透的脚本。

先查看Joomla版本


用searchspliot查询joomla该版本漏洞，发现sql注入漏洞


接下来就是用sqlmap跑库跑表

sqlmap -u “url” –dbms mysql –dbs

sqlmap -u “url” –dbms mysql -D joomladb –tables

sqlmap -u “url” –dbms mysql -D joolmladb -T ‘#__users’ –columns


sqlmap -u “url” –dbms mysql -D joomladb -T ‘#__users’ -C name,password –dump

得到hash加密过的密码

接下来用John破解密码
john the ripper是一款本地密码破解工具，可以从我们上面生成的shadow文件（密码散列）中破解出密码。破解时间取决于密码的复杂程度以及破解模式。

破解得到admin的密码为snoopy
登录网站发现啥也没有，猜测网站应该有后台，重新用dirsearch扫描得到/administrator,访问登录成功
