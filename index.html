kali实现局域网arp欺骗

2022-07-11
kali渗透

先确定kali（攻击机）和win7（靶机）处于同一网段

kali实现arp欺骗有两种方式

第一种，命令行用arpspoof进行arp攻击

先查看有没有arpspoof工具

没有则需要安装

apt-install arpspoof

提前打开wireshark，开始arp攻击

其中参数-i为指定端口，-t依次指定要欺骗的主机和伪装成的主机

查看wireshark所抓获的包，发现成功冒充为靶机的网关

回到win7我们发现已经无法访问网页了

回到kali终端Ctrl+z结束攻击，回到win7成功访问网页

第二种，用图像化工具ettercap

使用方法也很简单，打开sniffing，选择网卡

依次点击左上角的搜索框和列表，扫描并列举出存活主机

将需要欺骗的主机添加到target1，需要伪装的主机添加到target2

点击右上角的选项，选择第一个

勾选第一项后，开始攻击

成功欺骗

注：ettercap只是一个嗅探工具，并不会使靶机断网

拓展：ettercap与driftnet配合截取图片信息

安装driftnet：apt-install driftnet

进入driftnet

打开ettercap进行arp欺骗，在靶机浏览器访问网页

成功抓获图片

注：只能抓取http(未经加密)数据包的图片

linux用户管理

2022-07-02
kali

用户和组账号概述

用户账号 组账号 UID和GID
超级用户root 基本组（私有组） UID（用户标识号）
普通用户 附加组（公共组） GID（组标识号）
程序用户

用户账号文件–passwd

文件位置：/etc/passwd

格式：lifan(登录名): x(口令):1000(UID):1000(GID):lifan(注释):/home/lifan(主目录):/bin/bash(登录shell)

用户账号文件–shadow

文件位置：/etc/shadow

格式：lifan(用户账号的名称):$6$fqD3P4oEiS4Evv9Y$zVVsaz0PMbP8T6ZFX/TQIp/aDG9jViZwTvTiYuwI5lnQpahbKJM/B4QZbviEZBCEjrgyIKUQ1gLFAusJyOT.w.(加密密码):19174(最后一次修改时间):0(最小修改时间间隔):99999(密码有效期):7(密码需要变更前的警告天数): (密码过期后的宽限时间): (账号失效时间): (保留字段)

<–加密密码不能手工修改，否则系统会无法识别密码导致密码失效–>

<–最后一次修改时间的数值代表的是天–>

<–密码失效期默认值为99999，密码过期后的宽限时间、账号失效时间、一般默认为空、保留字段目前未使用–>

用户管理命令

useradd

格式：useradd [参数] 用户名

参数 作用
-u 指定UID
-d 指定宿主目录，默认为/home/用户名
-e 指定账号失效时间
-g 指定用户的基本组
-G 指定用户的附加组
-m 自动建立用户的登入目录(-M反之)
-s 指定用户是登录shell
userdel

格式：userdel [参数] 用户名

参数 作用
-f 强制删除用户，即使用户当前已登录
-r 删除用户的同时，删除与用户相关的所有文件
usermod

格式：usermod [参数] 用户名

参数 作用
-l 更改用户账号的登录名称
-L 锁定用户账户
-U 解锁用户账户
groupadd

格式：groupadd [参数] 用户组名

参数 作用
-g 指定新用户组的GID
-o 一般与-g选项同时使用，表示新用户组的GID可以与系统已有用户组的GID相同。
groupdel

格式：group 用户组名

groupmod

格式：groupmod [参数] 用户组名

参数 作用
-g 给群组指定一个新的GID，即修改组ID
-n 指定要更改的组名

<–修改用户组名字的前提是拥有该名字的组存在–>

设置/更改用户口令–passwd

格式：passwd [参数] 用户名

参数 作用
-d 清空用户的密码，无需密码即可登录
-l 锁定用户账号
-S 查看用户账号状态(是否被锁定)
-u 解锁用户账号

添加/删除组成员–gpasswd

格式：gpasswd [参数] 用户组名

参数 作用
-a 向组内添加一个用户
-d 从组内删除一个用户
-M 定义组成员列表，以逗号分离

用户和组账号查询

查询用户身份标识–id

格式：id [用户名]

查询用户所属组–groups

格式：groups [用户名]

查询用户账号的详细信息–finger

格式：finger [-l] [用户名]

查询已登录到主机的用户信息–users、w、who

图形化用户与组管理工具

文件/目录的权限和归属

访问权限 字符代表 数字代表
读取 r 4
写入 w 2
可执行 x 1
归属(所有权)
属主 拥有该文件或目录的用户账号
属组 拥有该文件或目录的组账号

设置文件/目录的权限–chmod

格式1：chmod [ugoa] [+-=] [rwx] 文件或目录

<–u、g、o、a分别表示属主、属组、其他用户、所有用户–>

<–+、-、=分别表示增加、去除、设置权限–>

<–r、w、x分别表示读取、写入、可执行–>

格式2：chmod [三位八进制数] 文件或目录

设置文件/目录的归属–chown

格式1：chown [属主] [文件或目录]

格式2：chown[:属组] [文件或目录]

格式3：chown [属主:属组] [文件或目录]

<–chmod和chown都可以用-R参数递归修改指定目录下所有文件、子目录的权限–>

特殊权限访问

SUID：设定时，文件执行将以文件拥有者身份执行，而非普通用户的身份

SGID：与前者相同，当文件被执行时，将以文件的组的身份来执行，而不是一般的用户身份

设置SUID：chmod u+s [文件]

​ chmod 4551 [文件]

设置SGID：chmod g+s [文件]

​ chmod 2551 [文件]

用户账号的初始配置文件

文件来源：新建用户账号是，从/etc/skel目录中复制而来

常见的用户初始配置文件 作用
~/.bash_profile 用户每次登录时执行
~/.bashrc 每次进入新的bash环境时执行
~/.bash_logout 用户每次退出登录时执行
