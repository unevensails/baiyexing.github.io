文件包含

2022-07-25
web安全

文件包含

漏洞产生原因

文件包含函数加载的参数没有经过过滤或者严格的定义，可以被用户控制，包含其他恶意文件，导致了执行了非预期的代码。

文件包含常用函数

include():找不到被包含的文件时会产生致命错误，并停止脚本运行

require():找不到被包含的错误时只会产生警告，脚本将继续运行

include_once():与include()类似，唯一区别是如果该文件中的代码已经被包含，则不会再次包含

require_once():与require()类似，唯一区别是如果该文件中的代码已经被包含，则不会再次包含

文件包含分类

本地文件包含

本地文件包含就是通过浏览器包含web服务器上的文件，这种漏洞是因为浏览器包含文件时没有进行严格过滤允许遍历目录的字符注入浏览器并执行

远程文件包含

如果PHP的配置选项allow_url_include、allow_url_fopen状态为ON的话，则include/require函数是可以加载远程文件的，这种漏洞被称为远程文件包含漏洞

注：本地文件包含与远程文件包含有着相同的原理，但前者只能包含本地服务器上存在的文件，而后者可以包含远程服务器上的文件

php伪协议

file://

zip://

访问压缩包里的文件

只需要是zip的压缩包即可，后缀名可以任意更改

phar://

与zip://类似，区别在phar中相对路径和绝对路径都可以使用
注：php版本需大于5.3

php://filter

读取指定文件源码
例如用base64加密读取phpinfo.php的源码

php://input

访问各个输入流
条件：allow_url_include :on

步骤:先访问该网址，然后bp抓包，最下面添加<?php phpinfo();?>，放包

注：除了data://和php://input，其他的伪协议都不受allow_url_fopen和allow_url_include控制

http协议

2022-07-12
web

web服务

C/S模型(客户/服务器模型)

http——超文本传输协议

端口：80

http允许传输任意类型的数据对象

无连接

限制每次连接只处理一个请求,处理完客户请求并收到客户的应答就断开连接

单向性

服务端永远被动等待客户端的请求

无状态

协议对事务处理没有记忆能力

注：为了解决无状态，出现了cookie和session

uri,url,urn之间的区别

uri

统一资源标识符

纯粹的句式结构

url

统一资源定位符

由传送协议，服务器(ip或者域名)，端口号，请求资源路径，传递数据五个元素

urn

统一资源名称

对比url，少了最前面的协议

urn是uri的子集

http请求方式

post,get,head,put,delete,trace

get只用于读取数据

head和get方法一样，不过服务器不传回资源的文本部分

post向指定资源提交数据

put向指定资源位置上传内容
delete请求服务器删除request-rui所标识的资源

get和post的区别

get请求只能进行url编码，post支持多种编码方式

get请求参数会被保留在浏览器历史记录，post参数不会被保留

get请求在url中传送的参数有长度限制，post没有

get比post更不安全，但get速度比post快

http请求报文

HTTP 请求报文由3部分组成：请求行+请求头+请求体

常见的HTTP报文头属性

请求方式

host 目标主机地址和端口

connection 维护客户端和服务端的连接关系

content-length http消息内容的传输长度

accept 发送端希望接受的数据类型

content-type 请求实体文档类型

referer 源地址

cookie

当客户端访问服务器的时候，服务器都会生成一份cookie传输给客户端，客户端会自动把cookie保存，以后客户端每次访问服务器，都会

自动的携带这份cookie

session比cookie安全，都是标识用户身份

http响应报文

响应报文与请求报文一样,由三个部分组成：响应行,响应头,响应体

http状态码

200 OK 表示成功

303 重定向,把你重定向到其他页面

304 资源并未修改,可以直接使用本地的缓存

404 找不到页面(页面被删除或其他)

500 服务端错误

响应头

Cache-Control 响应输出到客户端后,服务端通过该属性告诉客户端该怎么控制响应内容的缓存

ETag 表示你请求资源的版本,如果该资源发生了变化,那么这个属性也会跟着变

Location 在重定向中或者创建新资源时使用

Set-Cookie 服务端可以设置客户端的cookie
